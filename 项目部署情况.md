## 三、后端 SpringBoot 部署（Systemd 托管）

### 2. 打包命令
```bash

```
打包产物：`target/EduShare-0.0.1-SNAPSHOT.jar`

### 3. Systemd 服务配置
文件路径：`/etc/systemd/system/edushare.service`
```ini
[Unit]
Description=EduShare Backend Service
After=network.target mysql.service redis.service

[Service]
Type=simple
User=root
WorkingDirectory=/root/LearningResourceLibrary/target
ExecStart=/usr/bin/java -Xms256m -Xmx512m -jar EduShare-0.0.1-SNAPSHOT.jar
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

### 4. 后端运维命令

## 五、Nginx 反向代理 + HTTPS 配置
### 1. Nginx 运行用户修改（解决/root目录403）
文件：`/etc/nginx/nginx.conf`
首行修改：`user root;`

### 2. 站点配置文件
路径：`/etc/nginx/sites-available/edushare`
```nginx
server {
    listen 80;
    server_name 156.239.236.13 easydowork.cn www.easydowork.cn;
    # HTTP全部强制跳转HTTPS
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl;
    server_name 156.239.236.13 easydowork.cn www.easydowork.cn;

    # 允许上传最大20MB（Nginx默认1MB，超出会报413）
    client_max_body_size 20m;

    # Certbot自动生成ssl证书路径
    ssl_certificate /etc/letsencrypt/live/easydowork.cn/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/easydowork.cn/privkey.pem;

    # 前端静态页面
    location / {
        root /root/LearningResourceLibrary/frontend/dist;
        index index.html;
        # Vue history模式刷新404修复
        try_files $uri $uri/ /index.html;
    }

    # 后端API代理
    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 上传文件静态访问（单数，旧路径兼容）
    location /upload/ {
        alias /root/LearningResourceLibrary/upload/;
        expires 30d;
        autoindex off;
    }

    # 封面图片访问（复数带s，代码返回的URL路径）
    location /uploads/ {
        alias /root/LearningResourceLibrary/upload/;
        expires 30d;
        autoindex off;
    }
}
```

### 3. 站点启用 & 默认页面关闭
```bash
# 软链接启用站点
ln -s /etc/nginx/sites-available/edushare /etc/nginx/sites-enabled/
# 删除默认页面
unlink /etc/nginx/sites-enabled/default

# 校验配置 + 重载
nginx -t
systemctl reload nginx

# 开机自启（已启用）
systemctl enable nginx
```

## 六、HTTPS 免费SSL证书（Let’s Encrypt）
### 1. 安装工具
```bash
apt update
apt install certbot python3-certbot-nginx -y
```

### 2. 签发证书命令
```bash
certbot --nginx -d easydowork.cn -d www.easydowork.cn
```
交互选择：
1. 输入邮箱（接收到期提醒）
2. `Y` 同意服务条款
3. `N` 拒绝EFF邮件推送
4. 选择 `2` 强制HTTP跳转HTTPS

### 3. 证书自动续期定时任务
定时任务配置：`crontab -e`
```cron
# 每日凌晨3点自动检测续期，更新后重载Nginx
0 3 * * * certbot renew --quiet && systemctl reload nginx
```
验证定时任务：`crontab -l`

### 4. 手动测试续期
```bash
certbot renew --dry-run
```

## 八、代码更新全流程（长期维护用）
### 方式1：Git 一键更新脚本（推荐）
1. 创建更新脚本
```bash
nano /root/update.sh
```
```bash
#!/bin/bash
echo "===== 开始更新后端 ====="
cd /root/LearningResourceLibrary
git stash                         # 暂存服务器本地修改
git pull origin master
git stash drop                    # 不需要保留本地修改，直接丢弃
mvn clean package -DskipTests
systemctl restart edushare
echo "后端重启完成"

echo "===== 开始更新前端 ====="
cd /root/LearningResourceLibrary/frontend
git stash
git pull origin master
git stash drop
npm install
npm run build
systemctl reload nginx
echo "前端更新完成，全站更新完毕"
```
2. 添加执行权限
```bash
chmod +x /root/update.sh
```
3. 后续更新只执行：
```bash
/root/update.sh
```
