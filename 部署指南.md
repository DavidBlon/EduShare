## 一键更新（推荐）

服务器上已有 `/root/update.sh` 脚本，SSH 登录后直接执行：

```bash
ssh root@156.239.236.13
/root/update.sh
```

该脚本自动完成：`git pull` → 后端打包 → 后端重启 → 前端打包 → Nginx 重载。

## 仅部署前端

> 适用场景：只改了前端代码（Vue 组件、样式等），后端无变化。

```bash
# 1. SSH 登录服务器
ssh root@156.239.236.13

# 2. 进入前端目录并拉取最新代码
cd /root/LearningResourceLibrary/frontend
git pull origin master

# 3. 安装依赖（首次部署或 package.json 有变化时需要）
npm install

# 4. 生产打包
npm run build

# 5. 重载 Nginx 使新页面生效
systemctl reload nginx
```

**验证：** 浏览器访问 `https://easydowork.cn` 查看前端是否更新。

## 仅部署后端

> 适用场景：只改了后端代码（Java、SQL、配置等），前端无变化。

```bash
# 1. SSH 登录服务器
ssh root@156.239.236.13

# 2. 拉取最新代码并打包
cd /root/LearningResourceLibrary
git pull origin master
mvn clean package -DskipTests

# 3. 重启后端服务
systemctl restart edushare

# 4. 查看启动日志，确认启动成功
journalctl -u edushare -f --lines=50
```

**验证：** `curl http://localhost:8080/api/front/resource/list` 检查 API 是否正常返回。

## 前后端一起部署

> 适用场景：前后端都有改动，或者不确定改了哪些部分。

```bash
# 1. SSH 登录服务器
ssh root@156.239.236.13

# ===== 部署后端 =====
cd /root/LearningResourceLibrary
git pull origin master
mvn clean package -DskipTests
systemctl restart edushare
echo "后端部署完成"

# ===== 部署前端 =====
cd /root/LearningResourceLibrary/frontend
git pull origin master
npm install
npm run build
systemctl reload nginx
echo "前端部署完成"

echo "全站更新完毕"
```

## 常用命令速查

### 后端运维

```bash
systemctl status edushare        # 查看后端运行状态
systemctl restart edushare       # 重启后端
systemctl stop edushare          # 停止后端
systemctl start edushare         # 启动后端
journalctl -u edushare -f        # 实时查看后端日志
```

### Nginx 运维

```bash
systemctl status nginx           # 查看 Nginx 状态
systemctl reload nginx           # 重载配置（不影响现有连接）
systemctl restart nginx          # 重启 Nginx
nginx -t                         # 检查 Nginx 配置是否正确
```

### 数据库

```bash
mysql -u edushare -p edushare    # 登录数据库
```

### SSL 证书

```bash
certbot renew --dry-run          # 测试证书续期
crontab -l                       # 查看定时任务（证书每日自动续期）
```
